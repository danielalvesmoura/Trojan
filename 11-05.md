# Conceitos Fundamentais

## O que é Persistência

Persistência é o conceito de armazenar dados de forma permanente para que eles continuem disponíveis mesmo após o encerramento de um aplicativo ou desligamento do dispositivo.

Em aplicações, isso é essencial para salvar informações do usuário, configurações, autenticação, registros e qualquer dado que precise ser recuperado posteriormente.

Diferente dos dados mantidos apenas em memória durante a execução do programa, os dados persistidos ficam armazenados em bancos de dados, arquivos locais ou serviços na nuvem.

---

# Tipos de Persistência

## Local
É quando os dados são armazenados diretamente no próprio dispositivo do usuário, como no celular ou computador.

Esse tipo de armazenamento permite acesso rápido às informações e funcionamento mesmo sem internet, mas limita o compartilhamento e a atualização entre diferentes dispositivos.

## Remota
Ocorre quando os dados são armazenados em servidores externos, geralmente na nuvem.

Nesse caso, o acesso às informações depende de conexão com a internet, mas permite que os dados sejam consultados e atualizados de qualquer lugar.

## Sincronizada
Combina as duas abordagens, mantendo os dados tanto no dispositivo local quanto em um servidor remoto.

Dessa forma, o sistema pode funcionar offline e, quando a conexão é restabelecida, sincroniza automaticamente as informações entre o dispositivo e a nuvem.

---

# Funcionamento da Persistência em Aplicações Flutter

No Flutter, a persistência funciona por meio de diferentes mecanismos de armazenamento, escolhidos de acordo com a necessidade da aplicação.

Para dados simples, como preferências do usuário e configurações, é comum utilizar o `SharedPreferences`.

Para estruturas mais complexas e relacionais, podem ser usados bancos locais como:
- SQLite
- Hive

Também é possível integrar serviços em nuvem, como Firebase, permitindo sincronização entre dispositivos.

O Flutter acessa esses recursos utilizando pacotes específicos, que fazem a comunicação entre a interface do aplicativo e o sistema de armazenamento.

Assim, os dados podem ser gravados, consultados, atualizados e removidos conforme as ações do usuário.

---

# Sequência Lógica do Processo de Persistência

A sequência lógica do processo de persistência inicia quando o usuário realiza uma ação no aplicativo, como preencher um formulário ou alterar uma configuração.

Em seguida, o aplicativo coleta essas informações e as organiza em objetos ou estruturas de dados.

Depois disso, os dados são enviados para um mecanismo de armazenamento, onde ficam registrados de forma permanente.

Quando o aplicativo precisar dessas informações novamente, ocorre a leitura dos dados persistidos, permitindo sua recuperação e exibição na interface.

Caso haja alterações, os dados podem ser atualizados e salvos novamente, mantendo as informações sempre consistentes e disponíveis para futuras utilizações.
